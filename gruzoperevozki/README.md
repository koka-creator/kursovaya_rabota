# Система учета грузовых автоперевозок

Информационная система для поддержки деятельности автохозяйства, занимающегося междугородними грузовыми перевозками.

## Функциональность

Система позволяет управлять:

1. **Автомобилями** - регистрация парка автомобилей с характеристиками (гос. номер, марка, модель, грузоподъемность, назначение, год выпуска, год кап. ремонта, пробег, фотография)

2. **Водителями** - учет штата водителей (ФИО, табельный номер, год рождения, стаж, категория, классность)

3. **Клиентами** - регистрация физических и юридических лиц:
   - Физические лица: ФИО, телефон, паспортные данные
   - Юридические лица: название, ФИО руководителя, юридический адрес, телефон, банк, расчетный счет, ИНН

4. **Заказами** - оформление заказов на перевозку с указанием:
   - Дата заказа
   - Клиент-отправитель и адрес погрузки
   - Клиент-получатель и адрес разгрузки
   - Длина маршрута и стоимость
   - Перечень грузов (название, единица измерения, количество, вес, страховая стоимость)

5. **Рейсами** - планирование рейсов для выполнения заказов:
   - Выбор заказа
   - Выбор автомобиля
   - Формирование экипажа (один или несколько водителей)
   - Дата и время прибытия в пункт погрузки

## Требования

- .NET 6.0 SDK или выше
- Windows (приложение использует Windows Forms)

## Сборка проекта

### Сборка в один EXE файл

Для сборки приложения в один исполняемый файл выполните:

```bash
dotnet publish -c Release -r win-x64 --self-contained true /p:PublishSingleFile=true
```

Исполняемый файл будет находиться в папке:
```
bin\Release\net6.0-windows\win-x64\publish\Gruzoperevozki.exe
```

### Обычная сборка

Для обычной сборки проекта:

```bash
dotnet build
```

Запуск приложения:
```bash
dotnet run
```

## Хранение данных

Данные сохраняются в формате JSON в файле:
```
%AppData%\Gruzoperevozki\data.json
```

Все данные хранятся локально на компьютере пользователя.

## Использование

1. Запустите приложение
2. Используйте кнопки на главном экране для перехода к нужным разделам
3. В каждом разделе доступны операции:
   - Добавление новых записей
   - Редактирование существующих записей
   - Удаление записей
   - Просмотр списка записей

## Структура проекта

- `Models/` - классы данных (Car, Driver, Client, Order, Trip, CargoItem)
- `Forms/` - формы Windows Forms для пользовательского интерфейса
- `Data/` - классы для работы с хранением данных
- `Program.cs` - точка входа приложения

## Зависимости

- Newtonsoft.Json - для сериализации данных в JSON
